# v0.2 Clean Text Pipeline + Management UI (F7 + F9)

> Status: **COMPLETED** - 2026-02-10

## What Was Built

Deterministic 4-step text processing pipeline (Filler Removal → Custom Words → Snippets → Whitespace) plus management UI and CLI commands.

## Deliverables

### Core (MacParakeetCore)
- `CustomWord` model + `CustomWordRepository` (protocol + impl)
- `TextSnippet` model + `TextSnippetRepository` (protocol + impl)
- `TextProcessingPipeline` (pure struct, no side effects)
- `TextProcessingResult` (text + expanded snippet IDs)
- DB migrations: `v0.2-custom-words`, `v0.2-text-snippets`

### Service Integration
- `DictationService` — runs pipeline when mode != .raw, sets cleanTranscript
- `TranscriptionService` — same pattern
- `AppEnvironment` — wires repos + processingMode closure (defaults to .clean)
- `AppDelegate` — pastes cleanTranscript ?? rawTranscript

### Management UI
- `CustomWordsViewModel` + `CustomWordsView` (sheet from Settings)
- `TextSnippetsViewModel` + `TextSnippetsView` (sheet from Settings)
- `SettingsView` — Processing section with mode picker + manage buttons
- `SettingsViewModel` — processingMode, word/snippet counts

### CLI
- `macparakeet flow process "text"` — run pipeline
- `macparakeet flow words list/add/delete` — manage custom words
- `macparakeet flow snippets list/add/delete` — manage snippets
