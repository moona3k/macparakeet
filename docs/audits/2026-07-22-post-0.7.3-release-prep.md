## Verdict

The post-`v0.7.3` code is a release candidate after the repository-truth fixes
in this preparation branch. No new ADR is required: the recent meeting changes
implement ADR-025's accepted source-lifecycle, warning, reporting, and recovery
decisions rather than introducing a new architecture. The remaining ADR-025
VAD transcript-gap repair is explicitly deferred and is not represented as
shipped.

The public release remains `0.7.3` until an exact approved commit is built,
signed, notarized, installed, physically exercised, uploaded, and published
through Sparkle. This document separates repository/code readiness from those
ship operations.

## Repository Truth Fixed

1. `plans/active/2026-07-20-dapt-export.md` still said PR #854 was open after
   merge. It is now a completed implementation record.
2. The meeting health/artifact/speaker plan was fully implemented but remained
   listed as proposed. It is now archived; ADR-025 and the active mic-reliability
   plan own the remaining transcript-gap-repair work.
3. The mic-reliability plan still described live recovery and warning UI as
   unimplemented. It now records the exact #857/#860 boundary and leaves only
   the evidence-gated signal-inference and VAD repair remainder open.
4. The architecture diagram still counted seven export formats after DAPT
   became the eighth, and its protocol sketch/summary omitted several existing
   formats. All three architecture references now match the public service.
5. The release-channel blocks did not summarize the post-`0.7.3` candidate
   scope or distinguish it from the stable DMG.
6. The CLI had additive post-3.1.0 public behavior under the already-published
   `3.1.0` identity. The candidate is promoted to `3.2.0` in accordance with
   the CLI semver contract; the existing Homebrew formula remains on 3.1.0
   until matching 3.2.0 artifacts are published.
7. The distribution preflight did not explicitly stop an app release from
   bundling new CLI behavior under a previously published CLI version. The
   runbook now calls out that gate and the existing `CLIVersionTests` check.
8. `scripts/dev/ci_local.sh` forwarded frontend-only type-check budget flags
   directly to the Swift driver, which Swift 6.3.1 rejects before compilation.
   The script now uses the portable `-Xswiftc -Xfrontend -Xswiftc <flag>` form;
   the corrected form was proven with an optimized CLI-target build before the
   full local-CI gate was restarted.
9. The recently merged microphone configuration-change handler queried the
   stopped engine's input format for diagnostics before beginning recovery.
   Under parallel load that Core Audio query delayed recovery long enough to
   fail the recovery tests; in production it could delay the same route-change
   recovery it was meant to observe. Stopped engines now report an unavailable
   (`0`/`0`) diagnostic format and enter recovery without that stale hardware
   query. The affected test also guards its postcondition instead of indexing
   an undersized array after an expectation failure.

## Draft User-Facing Release Notes

### Highlights

- Choose the correct embedded audio track when a local media file contains
  more than one.
- Export transcripts as W3C DAPT alongside TXT, Markdown, subtitles, documents,
  and JSON.
- Copy completed meetings as one readable document containing the title,
  notes, and transcript.

### Reliability And Performance

- Meetings now recover microphone and system capture from more route and stream
  failures, show actionable source warnings, and preserve partial-capture facts
  for recovery and diagnostics.
- Long single-speaker transcripts render a bounded presentation view without
  truncating stored or exported transcript data.
- Immediate audio deletion no longer allows the retention sweeper to race the
  explicit post-transcription deletion path.
- Empty transcripts skip AI formatting instead of making a useless provider
  request.
